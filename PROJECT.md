# PROJECT.md — User Stories (VendorCompliance)

## Must-have (MVP)
1. As a compliance analyst, I want to upload a supplier PDF so that I can detect missing or invalid compliance data. (MUST)
2. As a reviewer, I want to view parsed supplier fields so that I can validate them. (MUST)
3. As a system, I want to store supplier documents securely so we can audit later. (MUST / NFR)

## Nice-to-have (post-MVP)
4. As a compliance analyst, I want to link a supplier document to a policy or requirement so that I can compare expected compliance. (Nice)
5. As a reviewer, I want to flag a supplier record for remediation so that procurement can hold approval. (Nice)
6. As an admin, I want logs for each upload so I can debug. (NF)
7. As a user, I want to download original PDF so I can share it internally or externally. (Nice)
8. As an analyst, I want basic metrics (documents processed, compliance gaps found) so I can measure impact. (Nice)

## Notes
- The first vertical slice (MVP) will implement stories 1–3 only.
